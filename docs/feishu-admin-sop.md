# PersEng CLI 飞书管理员接入 SOP

这份文档适合直接交给飞书管理员、IT 或运营同学执行。

目标：
- 为 `perseng-cli` 准备一个飞书机器人应用
- 安装到当前租户
- 让员工可在私聊/群聊里使用
- 支持管理员切换 6 个 Agent 角色

## 1. 你需要提前知道的事

这个项目的飞书接入方式是：
- 飞书机器人应用
- WebSocket 长连接
- 不依赖公网回调地址

这意味着：
- 不需要为飞书事件开放公网回调 URL
- 但需要一台能访问外网的服务器常驻运行机器人

## 2. 推荐接入方式

优先推荐使用项目自带命令：

```bash
node bin/perseng.js feishu-register
```

这个命令会输出一个授权链接。

管理员需要做的事：
- 打开链接
- 确认创建或授权应用
- 把拿到的 `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 发给部署同学

## 3. 管理员接入流程

### 3.1 获取应用凭据

有两种方式。

#### 方式 A：扫码/打开授权链接

由部署同学执行：

```bash
node bin/perseng.js feishu-register
```

管理员看到链接后：

1. 用浏览器打开
2. 确认创建或授权应用
3. 等待返回应用凭据

完成后部署同学会拿到：

```text
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=xxx
```

#### 方式 B：飞书开放平台手动创建

如果不走命令式注册，就在飞书开放平台手动创建企业自建应用。

管理员需要确保：
- 应用已创建
- 机器人能力已启用
- 应用已发布
- 应用已安装到当前租户

然后把 `App ID` / `App Secret` 提供给部署同学。

## 4. 安装到租户后的验证

管理员需要配合做首轮验收。

### 4.1 私聊验证

1. 在飞书里搜索并打开机器人
2. 直接发送一句测试消息
3. 确认机器人能正常回复

### 4.2 群聊验证

1. 把机器人拉进测试群
2. 在群里 `@机器人`
3. 发送一条测试消息
4. 确认机器人能正常回复

## 5. 角色切换规则

当前项目已经支持聊天内切换角色：

```text
/role list
/role show
/role set <roleId>
```

但注意：
- 只有白名单管理员才能执行 `/role set <roleId>`
- 群聊中角色按“人”隔离，不会互相污染

这意味着：
- A 在群里切到 `nuwa`，只影响 A 自己
- B 仍然可以保持 `jiangziya`

## 6. 管理员需要提供给部署同学的信息

如果要启用更稳的生产控制，管理员最好补这几项：

- 哪些用户可以切换角色
- 哪些群允许机器人响应
- 是否需要限制只有内部员工可用

部署时对应的配置是：

```env
PERSENG_FEISHU_ALLOW_USERS=
PERSENG_FEISHU_ALLOW_GROUPS=
PERSENG_FEISHU_ROLE_ADMINS=ou_xxx,ou_yyy
```

其中最重要的是：
- `PERSENG_FEISHU_ROLE_ADMINS`

## 7. 建议的首批角色使用方式

如果你们同一租户同一批人共用 1 个机器人，推荐做法是：

- 默认角色：`jiangziya`
- 需要特定能力时，由管理员或指定负责人切换角色
- 用 `/role list` 先确认角色清单

推荐管理员先整理一张角色说明表，例如：
- `jiangziya`：战略分析
- `nuwa`：角色设计
- `luban`：工具或工程实现

这样员工更容易理解什么时候切换。

## 8. 常见问题

### 8.1 为什么不需要公网回调地址

因为这里不是 HTTP 回调订阅，而是长连接模式。

### 8.2 为什么群里必须 `@机器人`

这是为了避免机器人在群里误触发，降低噪音。

### 8.3 为什么有的人能聊天但不能切换角色

因为机器人使用权限和角色切换权限可以分开配置。

常见做法是：
- 所有人都能用
- 只有管理员能切角色

### 8.4 如果机器人被拉进多个群怎么办

可以通过：

```env
PERSENG_FEISHU_ALLOW_GROUPS=oc_xxx,oc_yyy
```

限制只在指定群响应。

## 9. 管理员验收清单

管理员上线前建议逐项确认：

1. 应用已创建
2. 应用已安装到当前租户
3. 机器人能被搜索到或拉入群
4. 私聊机器人可回复
5. 群聊 `@机器人` 可回复
6. `/role list` 可用
7. 白名单管理员能 `/role set <roleId>`
8. 非白名单成员不能切换角色

## 10. 交付给部署同学的最少信息

管理员只要给部署同学这些内容即可：

- `FEISHU_APP_ID`
- `FEISHU_APP_SECRET`
- 角色切换管理员名单
- 需要放开的测试群或正式群名单
